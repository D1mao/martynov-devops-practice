## Условия запуска CI/CD

Для автоматизации сборки используется GitHub Actions.

Workflow запускается автоматически при следующих событиях:

* push в ветку `main`;
* создание или обновление Pull Request в ветку `main`.

Конфигурация workflow находится в файле:

```text
.github/workflows/build.yml
```

---

## Автоматическая сборка

При выполнении триггера GitHub Actions выполняет следующие шаги:

1. Клонирование репозитория.
2. Установка Node.js.
3. Установка зависимостей приложения через `npm ci`.
4. Сборка приложения командой `npm run build`.
5. Авторизация в GitHub Container Registry (GHCR).
6. Сборка Docker-образа приложения.
7. Публикация Docker-образа в GitHub Container Registry.
8. Отправка уведомления в Telegram о результате выполнения сборки.

В случае успешного выполнения workflow отправляется уведомление об успешной сборке.

В случае ошибки отправляется уведомление о неуспешной сборке.

---

## Ручная сборка

Сборка Docker-образа вручную выполняется командой:

```bash
docker build -t foodtracker:latest .
```

Локальный запуск контейнера:

```bash
docker run -d -p 8080:80 foodtracker:latest
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8080
```

---

## Хранение результатов сборки

Результатом работы workflow является Docker-образ приложения.

Собранные образы автоматически публикуются в GitHub Container Registry (GHCR).

Адрес хранилища:

```text
ghcr.io/<github_username>/foodtracker
```

Для данного проекта:

```text
ghcr.io/d1mao/foodtracker
```

Таким образом, результаты каждой успешной сборки сохраняются в контейнерном реестре GitHub и могут быть использованы для последующего развертывания приложения.

---

## Уведомления Telegram

Для информирования о ходе выполнения CI/CD используется Telegram Bot API.

При завершении workflow бот автоматически отправляет сообщение:

* об успешной сборке;
* либо об ошибке сборки.

Сообщение содержит информацию о репозитории, ветке и результате выполнения pipeline.
